  • In our programming 3 class we are learning object oriented programming (OOP). We are currently learning about inheritance and the reason why inheritance is so important is because it is one of the four pillars in programming. The four pillars are Encapsulation, Abstraction, Inheritance, and Polymorphism.
  • The reason why inheritance is so important is because it stores data in one class that can be passed to another class. It also allows us to reuse a lot of codes and organizes fields a lot better.
  • When creating different classes remember to always remember to set your: fields, constructor, and properties! In your class program remember to include the code for inheritance which is the semicolon between the two classes if it is going to be the child class.

For example:

public class Class B : Class A

  • Another thing to keep in mind is in your constructor you also need to include base and all the attributes from the inherited class in parentheses and the datatype and attribute name in method constructor followed by any new attributes added from this class.

For example:

public class Class B : Class A

public Class B(string firstName, string lastName, int managerID) : base(firstName, lastName)
